# autotranslate ‚Äî deterministic Swedish‚ÜíEnglish mirror of introprog

`autotranslate` builds the English side of the course material (`compendium-en/`, `slides-en/`)
from the Swedish source by translating LaTeX **without breaking the build**. It is real Scala
(`autotranslate/*.scala`, run via sbt or the `scratch/` programs), not a `.sc` script.

This README is for **both humans and agents**. If you only read one section, read
[When the English side has Swedish in it](#when-the-english-side-still-has-swedish) ‚Äî it tells you
which of the three clamp tools to reach for.

---

## 1. The core idea: determinism, never trust the model with markup

Translation is a pipeline with a strict **precedence order**. For every text unit, the first source
that has an answer wins:

```
OVERRIDE  >  authoritative (glossary)  >  cache  >  model
```

- **OVERRIDE** ‚Äî `Overrides.scala`, a compile-checked `Map[Swedish, English]` you (or an agent) curate.
  Highest precedence. Matched on the *clean* Swedish of a unit (markup masked out, then restored).
- **authoritative** ‚Äî bilingual glossary terms/explanations generated by the `glossary` project. Never
  model-translated.
- **cache** ‚Äî `translate-cache.tsv`, committed, so a clone regenerates the English side **model-free**
  (`--all` becomes all cache-hits). Reversible TSV encoding (`\`,`\n`,`\t`) so dense code/REPL units
  round-trip exactly.
- **model** ‚Äî last resort. A local/remote Ollama model (default backend = the *modly* GPU box,
  `gemma2:9b`). Temperature 0 + fixed seed for repeatability.

Around the model sits the safety machinery, because **the model must never see raw markup it could
destroy**:

1. **mask** ‚Äî a hand-rolled, brace-balanced tokenizer replaces anything fragile (code envs, math,
   commands, `\ifswedish‚Ä¶\fi`, inline emphasis) with placeholders. NOT regex, NOT a real parser.
2. **translate** the masked prose.
3. **restore** the placeholders.
4. **validate** ‚Äî reject a translation that introduced a LaTeX special, a paragraph break (`\n\n`) the
   source lacked, foreign-script chars, etc.
5. **Swedish-fallback** ‚Äî if validation fails, keep the original Swedish. A build that renders some
   Swedish is infinitely better than a build that doesn't compile.

> Design rule: any regex must be tested ‚Äî BR can't review regex by eye. Prefer the masking tokenizer
> and explicit env lists over clever patterns.

## 2. How to run it

All commands run from the introprog repo root. The autotranslator itself is the `autotranslate` sbt
sub-project (`autotranslateProject`); the `scratch/` programs are thin `scala-cli` launchers around the
same `Main` for ad-hoc runs without the sbt server.

| Goal | Command |
|------|---------|
| Translate everything (both mirrors), via sbt | `sbt "autotranslateProject/run --all"` (or the alias `sbt autotranslate` = no args) |
| ‚Ä¶only one week | `sbt "autotranslateProject/run --only w01"` |
| Translate everything, via scratch (no sbt server) | `scala-cli run autotranslate/scratch/at.scala -- . --all --model gemma2:9b` |
| Build one English deck | `scala-cli run autotranslate/scratch/build-deck.scala -- . w05` |
| Build all 19 English decks | `scala-cli run autotranslate/scratch/slides-en.scala -- .` |
| English compendium halves (fast checks) | `sbt --client pdfCompendium1En` / `pdfCompendium2En` |
| Full English compendium (incl. pre/postchapters) | `sbt --client pdfCompendiumEn` |
| Swedish builds (verify a source edit didn't break sv) | `sbt --client pdfSlides` / `pdfCompendium` |
| Corpus Swedish gauge (model-free, read-only) | `... at.scala -- . --swedish-left` |
| Swedish prose lines in ONE en file | `... at.scala -- . --swedish-lines slides-en/body/lect-w05-classes-en.tex` |
| Exact override keys for still-Swedish units | `... at.scala -- . --all --sweep-fallbacks` ‚Üí `scratch/override-suggestions.txt` |

A clean run reports `done. model calls: 0` ‚Äî the committed cache + overrides cover the whole corpus, so
no network/GPU is needed to regenerate. A non-zero count means a unit changed keys (e.g. you edited
masking) and is being re-translated.

`autotranslateProject/run` flags (same `Main`, whether via sbt or `scratch/at.scala`): `--all`,
`--only <wNN>`, `--model <name>`, `--swedish-left`, `--swedish-lines <file>`, `--pdf-swedish <pdf>`,
`--sweep-fallbacks`, `--dump-overrides`, `--clean`. The sbt form defaults the repo root; the scratch
form takes it as the first arg (`.`).

### 2a. The sbt `build` task builds the **Swedish** side only (this is what CI runs)

The PDF build tasks come in two families:

- **Swedish** (the source language): `pdf`, `pdfCompendium`, `pdfCompendium1`/`2`, `pdfSlides`,
  `pdfExercises`, ‚Ä¶ and the umbrella command alias **`build`** =
  `addCommandAlias("build", "gen; genquiz; gengloss; pdf; gen")`. **`sbt build` does NOT touch the
  English mirror** ‚Äî it runs the generator projects (plan/quiz/glossary) then `pdf` (the Swedish PDFs).
- **English** (the generated mirror): `pdfCompendiumEn`, `pdfCompendium1En`/`2En`, `pdfSlidesEn`. These
  read `compendium-en/`, `slides-en/`, so you must run the translator (`autotranslateProject/run --all`)
  first to (re)generate the mirror.

**CI** (`.github/workflows/main.yml`) runs `sbt build` only ‚Üí it compiles and builds the **Swedish**
material and never runs autotranslate or the `*En` tasks. So the English side is **not** gated by CI; it
is regenerated model-free from the committed `translate-cache.tsv` on demand (and published separately).
Practical consequence: a translation/clamp change cannot break the CI build, but you must verify the
English build yourself (see ¬ß4) ‚Äî CI won't catch an English-only breakage.

## 3. When the English side still has Swedish in it

Pick the tool by **what kind of thing** is still Swedish:

| The Swedish is‚Ä¶ | Use | Why |
|-----------------|-----|-----|
| **Clean prose / a proper noun** the model echoed or mangled | **`Overrides.scala`** | A human/agent translates better than gemma2; highest precedence; one entry fixes every occurrence. |
| **Inside a figure** (`tikzpicture`, `pgfpicture`, ‚Ä¶) | **`\ifswedish` in the source node** | Figure envs are masked whole ‚Üí never translated. The only way in is a source clamp. |
| **A one-off pathological construct** the masker/model keeps breaking | **`\ifswedish` in the source** | Author escape hatch; you hand-write the English once. |
| **A recurring non-prose environment** (pseudocode, a custom verbatim env) | add it to **`verbatimEnvs`** in `Latex.scala` | Mask the whole env so its body is never translated (accepted residual, like code). |

### 3a. Overrides.scala ‚Äî the reliable workflow

Blind keys guessed from a rendered `-en` line usually **don't match** (the translator's clean key differs
from what you see). Always get the **exact** key from the sweep:

```
# 1. regenerate the exact-key list for every still-Swedish unit, all files
scala-cli run autotranslate/scratch/at.scala -- . --all --sweep-fallbacks
#    -> scratch/override-suggestions.txt, grouped:  [N] sv-fallback | <file>
#                                                    SV |<exact clean key>
#                                                    EN |<current Swedish output>

# 2. pull the keys for a target file
grep -A1 'sv-fallback | compendium/modules/w07-sequences-exercise' scratch/override-suggestions.txt | grep 'SV |'
```

Then add to `Overrides.scala` (triple-quoted, literal backslashes; keep the SAME `\Emph{}`/`\code{}`
structure as the key):

```scala
"""En \Emph{variabel} kan tilldelas v√§rdet av ett enkelt eller sammansatt uttryck.""" ->
  """A \Emph{variable} can be assigned the value of a simple or compound expression.""",
```

Verify: re-run `--all` and check the `overrides:` **hit count** rose by your batch size (that confirms
the keys matched), then build the affected target and commit.

Notes & traps:
- Skip multi-line units (shown with `‚èé` in the sweep) and pure-`\code{}`/code-block units (accepted
  code residual) unless you really want them.
- A line whose only Swedish is a **`\code{}` identifier** (`po√§ng`, `F√§rg`, `Gurka`) still counts as
  "Swedish" in the gauge even after the prose is English ‚Äî so the % moves slowly; the reader-facing
  prose is what actually improves.
- Triple-quoted keys handle embedded `"` fine; just avoid a literal `"""` sequence.

### 3b. `\ifswedish ‚Ä¶ \else ‚Ä¶ \fi` ‚Äî the source clamp

Defined next to `\ifkompendium`. The tokenizer masks the whole block **verbatim** (it is never sent to
the model), the Swedish build sets `\swedishtrue` (shows the `\if` branch), the English build sets
`\swedishfalse` (shows the `\else` branch). Use it when translation can't reach the text or keeps
breaking it.

**Figures (tikz) ‚Äî the common case.** Clamp inside each prose node:

```latex
\node (input) [startstop] {\ifswedish K√§llkod\else Source code\fi};
```

- Clamp **prose** nodes; **leave code-identifier** nodes (`\texttt{√•lder}`, `\code{Gr√∂nsak}`) as code.
- For a mixed node, clamp the whole content: `{\ifswedish\texttt{.class}-fil med bytekod\else\texttt{.class} file with bytecode\fi}`.
- `\ifswedish` works inside tikz node text and tabular cells ‚Äî verified.
- The same figure is often duplicated across files (e.g. the compiler diagram in `lect-w01-intro`,
  `slides/simple/intro`, `postchapters/compile`) ‚Äî fix all copies.
- Helper to find them: `scala-cli run autotranslate/scratch/scan-tikz-swedish.scala -- .` lists Swedish
  inside tikz/pgf envs (note: it catches √•√§√∂ + a Swedish-CS-term list, but **read each block fully** ‚Äî short
  Swedish labels without diacritics can still be missed).

**If the English shouldn't differ** (e.g. pseudocode that must stay as-is): make `\else` identical to the
`\if` branch and flag it for BR rather than inventing a translation.

### 3c. verbatimEnvs ‚Äî mask a whole environment

In `Latex.scala`, `verbatimEnvs` lists environments masked as a single opaque unit (code, REPL,
`tikzpicture`, `algorithm`, `exlatex`, `envi`, ‚Ä¶). Add an env here when a recurring non-prose
environment is leaking into translation and getting mangled. This is a global, structural fix ‚Äî prefer
it over many one-off `\ifswedish` clamps when the same env type recurs.

## 4. Verification discipline (do not skip)

Translation/masking changes can break the build in ways slides-only checks miss. Match the check to the
change:

- Edited **w01‚Äìw07** content ‚Üí fast check `pdfCompendium1En`.
- Edited **w08‚Äìw14** content ‚Üí fast check `pdfCompendium2En`.
- Edited a **deck** ‚Üí `build-deck <wNN>` or `slides-en` (all 19).
- **Cross-cutting change** (masking rule, authoritative term, a `\ifswedish`/tikz source pass, a new
  baseline) ‚Üí run **all three**: `pdfCompendium1En` + `pdfCompendium2En` + `pdfCompendiumEn`, plus
  `slides-en`. Pre/postchapters (appendices) live only in the **full** compendium, so 1En/2En will not
  catch them.
- A change to **shared source** (e.g. a `\ifswedish` clamp) affects **both** languages ‚Üí also build the
  **Swedish** side (`pdfSlides` / `pdfCompendium`) to confirm you didn't break the `\if` branch.

A clean English build prints `[swedish] <file>.pdf: N% Swedish` and produces the PDF; a break shows
`Undefined control sequence` / `Runaway argument` / `Fatal error`.

## 5. Gotchas

- **`sbt --client` uses a persistent server.** To stop a running task, kill the server JVM
  (`--detach-stdio`), not the client. After editing `build.sbt`, run `sbt --client reload`.
- **Masking changes force a re-translate.** Changing what gets masked re-keys units ‚Üí a one-time model
  pass (minutes on the GPU box) before `--all` converges back to 0 calls. Watch `scratch/progress.txt`.
- **Don't git-track the generated `-en` side.** `compendium-en/`, `slides-en/` (`.tex` + PDFs) are
  gitignored; clones regenerate model-free from the committed cache. Readers get PDFs via `publish.sh`
  (fileadmin) and the GitHub release.
- **Course name** in English is "Introduction to programming"; **LTH** in English is "Lund University,
  Faculty of Engineering (LTH)".
- The cover and `workspace-en` are **static committed mirrors**, not autotranslated ‚Äî edit them by hand.

## 7. Internals: the masking tokenizer (for reviewers)

This section is for an experienced programmer reviewing `Latex.scala` / `Translate.scala`. The hard
problem is: *send the model only translatable prose, then put it back exactly, on live LaTeX, without a
parser*. The whole design follows from "never trust the model with markup, and never risk the round-trip".

### 7.1 Representation: placeholders + a spans vector

`Latex.mask(text, stripEng, collapseEmph=false): (String, IndexedSeq[String], Set[Int])` returns:

- a **masked string** in which every fragile fragment has been replaced by a token `__C<n>__` (regex
  `Place = __C(\d+)__`), leaving only translatable prose between tokens;
- **`spans`** ‚Äî an `IndexedSeq[String]` where `spans(n)` is the verbatim original of token `__C<n>__`;
- **`itemIdx`** ‚Äî the set of span indices that are `\item`-like (list-segmentation boundaries, ¬ß7.4).

`protect(span)` is the one primitive: it appends `__C${spans.size}__` to the output and pushes `span`.
Because tokens are numbered by emission order, `restore(masked, spans)` is a single linear pass that
swaps each `__C<n>__` back for `spans(n)` (unknown indices are left as-is ‚Äî defensive). Round-trip is
therefore exact *by construction*: the model only ever rewrites the prose between opaque tokens.

### 7.2 The scan: one linear, brace-balanced pass ‚Äî not regex, not a parser

`mask` walks the string with a single index `i` and a hand-rolled, brace-balanced scanner. Deliberate
choice: a recursive-descent parser risks losing bytes on the round-trip; broad regex is unreviewable
(BR's rule: regex must be tested, not eyeballed). The scan dispatches on what is at `i`:

- **newline** ‚Üí emitted as text (paragraph structure must survive; a model that introduces a `\n\n` the
  source lacked is rejected downstream ‚Äî see 7.5).
- **inline/display math** `$‚Ä¶$`, `\[‚Ä¶\]`, `\(‚Ä¶\)` ‚Üí the whole span is `protect`ed.
- **LaTeX specials** `\\`, `\{`, `\%`, `\$`, `\,`, ‚Ä¶ ‚Üí `protect`ed as a token, so they are preserved
  verbatim *and* participate in order-validation.
- **`\begin{env}`** ‚Üí if `env ‚àà verbatimEnvs` (Code/REPL/tikzpicture/algorithm/‚Ä¶) the **whole
  environment** (to its matching `\end{env}`) is one span; otherwise the `\begin`/`\end` markers are
  masked and the body between them stays translatable.
- **`\ifswedish`** ‚Üí `matchFi` scans to the matching `\fi` (nesting-aware: it counts inner `\if‚Ä¶`/`\fi`),
  and the whole `\ifswedish ‚Ä¶ [\else ‚Ä¶] \fi` block is one verbatim span. This is exactly why the author
  escape hatch (¬ß3b) is untouchable by the model.
- **control words** (`\cmd`) dispatch by name via the category sets:
  - `maskWhole` ‚Üí command **and its argument(s)** masked as one span (e.g. refs, includes, listings);
  - `itemCmds` (`item`/`ii`/`is`/`di`) ‚Üí mask the marker (+ optional `[..]`), and record the span index
    in `itemIdx`;
  - `headingCmds` / other **TRANSLATE-ARG** commands ‚Üí mask the `\cmd{` and the closing `}` as separate
    tokens but **leave the argument text exposed** for translation;
  - `emphArg` (`Emph`/`Alert`/`emph`/`textbf`/‚Ä¶) ‚Üí see 7.3;
  - zero-arg / font-switch / unknown control word ‚Üí mask just the control word, prose continues.

Helpers: `skipGroup` (balanced `{‚Ä¶}`), `skipOptional` (`[‚Ä¶]`), `groupContent` (returns arg + end index),
`matchFi`, `isCmdLetter`. All are index-returning and total (they clamp at end-of-string), so the scan
cannot run off the end or desynchronize.

### 7.3 Two-tier inline-emphasis masking (the dominant fallback fix)

Inline emphasis is the hard case: `‚Ä¶ \Emph{x} ‚Ä¶ \Alert{y} ‚Ä¶`. If each `\cmd{arg}` is masked as
command+`{`+text+`}`, a single bullet becomes *placeholder-dense* (3 tokens per emphasis), and small
models drop or reorder placeholders ‚Üí the unit fails validation ‚Üí Swedish fallback.

- **Tier 1 (default, `collapseEmph=false`):** the emphasis **argument text stays exposed** (only the
  `\cmd{`/`}` wrappers are tokens). The model sees natural prose with the emphasized words in context ‚Äî
  best grammar, reuses the existing cache.
- **Tier 2 (`collapseEmph=true`):** triggered by `Translate` *only* when a tier-1 result still looks
  Swedish **and** `hasEmphArg(clean)` is true. Now each `\cmd{arg}` collapses to **one** token, and the
  inner `arg` is translated as a child unit via `reEmphArg(span, translate)`. One placeholder per
  emphasis instead of three ‚Üí far fewer drops on the placeholder-dense bullets that tier 1 missed.

### 7.4 Segmentation: `segmentMasked`

`segmentMasked(masked, itemIdx)` splits the masked text into translation **units** so each model call is
small: it cuts just before every `\item` token (using `itemIdx`), and then **sentence-splits any "dense"
base block** (more than `DenseThreshold = 3` placeholders) on sentence boundaries. Fewer placeholders per
unit ‚áí dramatically fewer model placeholder-drops. It returns the units plus the separators needed to
reassemble them losslessly.

### 7.5 Validation & fallback (`Translate.invalidReason`)

After restore, a candidate translation is rejected (‚Üí keep Swedish) if it: drops/reorders the masked
tokens or LaTeX specials; introduces a LaTeX special the source lacked; introduces a paragraph break
(`\n\n`) absent from the source (the gemma2 `Latex:`‚Üí`Latex:\n\nEnglish:` hallucination that broke
lists); or contains foreign-script characters. The committed cache is trusted for build-*safe*
guard-flags (too-long / placeholder-reorder) but build-*breaking* flags are dropped on load so they
re-translate. Net rule restated: **a build that renders some Swedish always beats one that doesn't
compile.**

### 7.6 What a reviewer should poke at

- `mask`'s control-word dispatch (the big `match`) ‚Äî does every branch advance `i` past the construct?
- `matchFi` nesting and `skipGroup`/`skipOptional` boundary clamping (off-by-one / EOF).
- `reEmphArg` ‚Äî does it reconstruct `\cmd{translated}` for exactly the `emphArg` commands and pass other
  spans through untouched?
- `invalidReason` ‚Äî is the placeholder/specials order-check symmetric (no false accept that would let a
  reordered span through)?
- `enChrome` (`Main.scala`) ‚Äî literal `.replace` over each mirror; order-sensitive for overlapping keys
  (e.g. the `Resource Time`‚Üí`Tutorial` + trailing-`s` plural trick in ¬ß3/strand-2).
