# autotranslate — deterministic Swedish→English mirror of introprog

`autotranslate` builds the English side of the course material (`compendium-en/`, `slides-en/`)
from the Swedish source by translating LaTeX **without breaking the build**. It is real Scala
(`autotranslate/*.scala`, run via sbt or the `scratch/` programs), not a `.sc` script.

This README is for **both humans and agents**. If you only read one section, read
[When the English side has Swedish in it](#when-the-english-side-still-has-swedish) — it tells you
which of the three clamp tools to reach for.

---

## 1. The core idea: determinism, never trust the model with markup

Translation is a pipeline with a strict **precedence order**. For every text unit, the first source
that has an answer wins:

```
OVERRIDE  >  authoritative (glossary)  >  cache  >  model
```

- **OVERRIDE** — `Overrides.scala`, a compile-checked `Map[Swedish, English]` you (or an agent) curate.
  Highest precedence. Matched on the *clean* Swedish of a unit (markup masked out, then restored).
- **authoritative** — bilingual glossary terms/explanations generated by the `glossary` project. Never
  model-translated.
- **cache** — `translate-cache.tsv`, committed, so a clone regenerates the English side **model-free**
  (`--all` becomes all cache-hits). Reversible TSV encoding (`\`,`\n`,`\t`) so dense code/REPL units
  round-trip exactly.
- **model** — last resort. A local/remote Ollama model (default backend = the *modly* GPU box,
  `gemma2:9b`). Temperature 0 + fixed seed for repeatability.

Around the model sits the safety machinery, because **the model must never see raw markup it could
destroy**:

1. **mask** — a hand-rolled, brace-balanced tokenizer replaces anything fragile (code envs, math,
   commands, `\ifswedish…\fi`, inline emphasis) with placeholders. NOT regex, NOT a real parser.
2. **translate** the masked prose.
3. **restore** the placeholders.
4. **validate** — reject a translation that introduced a LaTeX special, a paragraph break (`\n\n`) the
   source lacked, foreign-script chars, etc.
5. **Swedish-fallback** — if validation fails, keep the original Swedish. A build that renders some
   Swedish is infinitely better than a build that doesn't compile.

> Design rule: any regex must be tested — BR can't review regex by eye. Prefer the masking tokenizer
> and explicit env lists over clever patterns.

## 2. How to run it

All commands run from the introprog repo root. The `scratch/` programs are launched with `scala-cli`.

| Goal | Command |
|------|---------|
| Translate everything (both mirrors) | `scala-cli run autotranslate/scratch/at.scala -- . --all --model gemma2:9b` |
| Build one English deck | `scala-cli run autotranslate/scratch/build-deck.scala -- . w05` |
| Build all 19 English decks | `scala-cli run autotranslate/scratch/slides-en.scala -- .` |
| English compendium halves (fast checks) | `sbt --client pdfCompendium1En` / `pdfCompendium2En` |
| Full English compendium (incl. pre/postchapters) | `sbt --client pdfCompendiumEn` |
| Swedish builds (verify a source edit didn't break sv) | `sbt --client pdfSlides` / `pdfCompendium` |
| Corpus Swedish gauge (model-free, read-only) | `... at.scala -- . --swedish-left` |
| Swedish prose lines in ONE en file | `... at.scala -- . --swedish-lines slides-en/body/lect-w05-classes-en.tex` |
| Exact override keys for still-Swedish units | `... at.scala -- . --all --sweep-fallbacks` → `scratch/override-suggestions.txt` |

A clean run reports `done. model calls: 0` — the committed cache + overrides cover the whole corpus, so
no network/GPU is needed to regenerate. A non-zero count means a unit changed keys (e.g. you edited
masking) and is being re-translated.

## 3. When the English side still has Swedish in it

Pick the tool by **what kind of thing** is still Swedish:

| The Swedish is… | Use | Why |
|-----------------|-----|-----|
| **Clean prose / a proper noun** the model echoed or mangled | **`Overrides.scala`** | A human/agent translates better than gemma2; highest precedence; one entry fixes every occurrence. |
| **Inside a figure** (`tikzpicture`, `pgfpicture`, …) | **`\ifswedish` in the source node** | Figure envs are masked whole → never translated. The only way in is a source clamp. |
| **A one-off pathological construct** the masker/model keeps breaking | **`\ifswedish` in the source** | Author escape hatch; you hand-write the English once. |
| **A recurring non-prose environment** (pseudocode, a custom verbatim env) | add it to **`verbatimEnvs`** in `Latex.scala` | Mask the whole env so its body is never translated (accepted residual, like code). |

### 3a. Overrides.scala — the reliable workflow

Blind keys guessed from a rendered `-en` line usually **don't match** (the translator's clean key differs
from what you see). Always get the **exact** key from the sweep:

```
# 1. regenerate the exact-key list for every still-Swedish unit, all files
scala-cli run autotranslate/scratch/at.scala -- . --all --sweep-fallbacks
#    -> scratch/override-suggestions.txt, grouped:  [N] sv-fallback | <file>
#                                                    SV |<exact clean key>
#                                                    EN |<current Swedish output>

# 2. pull the keys for a target file
grep -A1 'sv-fallback | compendium/modules/w07-sequences-exercise' scratch/override-suggestions.txt | grep 'SV |'
```

Then add to `Overrides.scala` (triple-quoted, literal backslashes; keep the SAME `\Emph{}`/`\code{}`
structure as the key):

```scala
"""En \Emph{variabel} kan tilldelas värdet av ett enkelt eller sammansatt uttryck.""" ->
  """A \Emph{variable} can be assigned the value of a simple or compound expression.""",
```

Verify: re-run `--all` and check the `overrides:` **hit count** rose by your batch size (that confirms
the keys matched), then build the affected target and commit.

Notes & traps:
- Skip multi-line units (shown with `⏎` in the sweep) and pure-`\code{}`/code-block units (accepted
  code residual) unless you really want them.
- A line whose only Swedish is a **`\code{}` identifier** (`poäng`, `Färg`, `Gurka`) still counts as
  "Swedish" in the gauge even after the prose is English — so the % moves slowly; the reader-facing
  prose is what actually improves.
- Triple-quoted keys handle embedded `"` fine; just avoid a literal `"""` sequence.

### 3b. `\ifswedish … \else … \fi` — the source clamp

Defined next to `\ifkompendium`. The tokenizer masks the whole block **verbatim** (it is never sent to
the model), the Swedish build sets `\swedishtrue` (shows the `\if` branch), the English build sets
`\swedishfalse` (shows the `\else` branch). Use it when translation can't reach the text or keeps
breaking it.

**Figures (tikz) — the common case.** Clamp inside each prose node:

```latex
\node (input) [startstop] {\ifswedish Källkod\else Source code\fi};
```

- Clamp **prose** nodes; **leave code-identifier** nodes (`\texttt{ålder}`, `\code{Grönsak}`) as code.
- For a mixed node, clamp the whole content: `{\ifswedish\texttt{.class}-fil med bytekod\else\texttt{.class} file with bytecode\fi}`.
- `\ifswedish` works inside tikz node text and tabular cells — verified.
- The same figure is often duplicated across files (e.g. the compiler diagram in `lect-w01-intro`,
  `slides/simple/intro`, `postchapters/compile`) — fix all copies.
- Helper to find them: `scala-cli run autotranslate/scratch/scan-tikz-swedish.scala -- .` lists Swedish
  inside tikz/pgf envs (note: it catches åäö + a Swedish-CS-term list, but **read each block fully** — short
  Swedish labels without diacritics can still be missed).

**If the English shouldn't differ** (e.g. pseudocode that must stay as-is): make `\else` identical to the
`\if` branch and flag it for BR rather than inventing a translation.

### 3c. verbatimEnvs — mask a whole environment

In `Latex.scala`, `verbatimEnvs` lists environments masked as a single opaque unit (code, REPL,
`tikzpicture`, `algorithm`, `exlatex`, `envi`, …). Add an env here when a recurring non-prose
environment is leaking into translation and getting mangled. This is a global, structural fix — prefer
it over many one-off `\ifswedish` clamps when the same env type recurs.

## 4. Verification discipline (do not skip)

Translation/masking changes can break the build in ways slides-only checks miss. Match the check to the
change:

- Edited **w01–w07** content → fast check `pdfCompendium1En`.
- Edited **w08–w14** content → fast check `pdfCompendium2En`.
- Edited a **deck** → `build-deck <wNN>` or `slides-en` (all 19).
- **Cross-cutting change** (masking rule, authoritative term, a `\ifswedish`/tikz source pass, a new
  baseline) → run **all three**: `pdfCompendium1En` + `pdfCompendium2En` + `pdfCompendiumEn`, plus
  `slides-en`. Pre/postchapters (appendices) live only in the **full** compendium, so 1En/2En will not
  catch them.
- A change to **shared source** (e.g. a `\ifswedish` clamp) affects **both** languages → also build the
  **Swedish** side (`pdfSlides` / `pdfCompendium`) to confirm you didn't break the `\if` branch.

A clean English build prints `[swedish] <file>.pdf: N% Swedish` and produces the PDF; a break shows
`Undefined control sequence` / `Runaway argument` / `Fatal error`.

## 5. Gotchas

- **`sbt --client` uses a persistent server.** To stop a running task, kill the server JVM
  (`--detach-stdio`), not the client. After editing `build.sbt`, run `sbt --client reload`.
- **Masking changes force a re-translate.** Changing what gets masked re-keys units → a one-time model
  pass (minutes on the GPU box) before `--all` converges back to 0 calls. Watch `scratch/progress.txt`.
- **Don't git-track the generated `-en` side.** `compendium-en/`, `slides-en/` (`.tex` + PDFs) are
  gitignored; clones regenerate model-free from the committed cache. Readers get PDFs via `publish.sh`
  (fileadmin) and the GitHub release.
- **Course name** in English is "Introduction to programming"; **LTH** in English is "Lund University,
  Faculty of Engineering (LTH)".
- The cover and `workspace-en` are **static committed mirrors**, not autotranslated — edit them by hand.
